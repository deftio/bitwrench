## Must-Have (paradigm correctness)

These fix bugs or wire up incomplete features. Ship or defer with reason.

### 1. Slot target caching (bug fix) -- DONE

Slot setters now cache the target node at creation time in the IIFE closure.
No more `el.querySelector(selector)` on every invocation.

Files changed: `src/bitwrench.js` (createDOM, slot setter generation)
Tests: 4 tests in `test/bitwrench_test_new_apis.js`

### 2. o.type wiring -- DONE

`if (opts.type) el._bw_type = opts.type` in createDOM. Enables `bw.inspect()`
and debug tooling to report component types.

Files changed: `src/bitwrench.js` (createDOM)
Tests: 4 tests in `test/bitwrench_test_new_apis.js`

### 3. Error boundaries in lifecycle callbacks -- DONE

`o.mounted`, `o.render`, and `o.unmount` wrapped in try/catch. Errors logged
via `_cw()` (console.warn) instead of crashing the component. Matches the
pattern already used in `bw.pub()`.

Files changed: `src/bitwrench.js` (createDOM mounted, update render, cleanup unmount)
Tests: 6 tests in `test/bitwrench_test_new_apis.js`

### 4. bw.inspect() in bitwrench.js core -- DONE

Rewritten from scratch. Returns plain object with bitwrench metadata: tag, id,
uuid, type, handles, state keys, sub count, hasRender, refs, lifecycle status.
Recursive walk with configurable depth (default 3). Old bw.inspect() only
logged to console with no useful return value.

Breaking change: new signature `bw.inspect(target, depth)` returns plain object.

Files changed: `src/bitwrench.js`, `src/bitwrench.d.ts`
Tests: 17 tests in `test/bitwrench_test_new_apis.js`

### 5. Enhanced _bw_tree in bwclient.js -- DONE

`_bw_tree` builtin now delegates to `bw.inspect()` when available (checks
`bw.inspect.length === 2` to distinguish new 2-arg from hypothetical old).
Falls back to hand-rolled walker for older bitwrench versions.

Files changed: `src/bwserve/bwclient.js`

### 6. client.inspect() + /inspect CLI rename -- DONE

Added `client.inspect(selector?, opts?)` to BwServeClient -- calls `_bw_tree`
builtin, returns promise with tree object. Matches `client.screenshot()` and
`client.query()` patterns. Renamed `/tree` to `/inspect` in bwcli attach REPL;
`/tree` kept as alias.

Files changed: `src/bwserve/client.js`, `src/cli/attach.js`
Tests: 6 tests in `test/bitwrench_test_attach.js`

### 9. bw.once(topic, handler) -- DONE

One-shot subscription. Auto-unsubscribes after first fire. Returns cancel
function.

Files changed: `src/bitwrench.js` (near bw.sub)
Tests: 10 tests in `test/bitwrench_test_new_apis.js`

### 10. Wildcard / namespace subscriptions -- DONE

`bw.sub('agui:*', handler)` -- trailing `*` matches any topic starting with
the prefix before `*`. Handler receives `(detail, topic)` so it can
distinguish which topic fired. Bare `'*'` matches all topics. Works with
`bw.once()`, element lifecycle, `bw.unsub()`. Publishing the pattern string
itself (e.g. `bw.pub('foo:*')`) fires only the exact-match subscriber, not
the wildcard path (no double-fire).

Files changed: `src/bitwrench.js` (bw.pub rewritten, bw.sub JSDoc updated),
`src/bitwrench.d.ts`
Tests: 15 tests in `test/bitwrench_test_new_apis.js`

### 11. bw.catalog() -- DONE

Exposes BCCL component registry. No args returns array of all types; with
type arg returns single entry or null. Uses `def.make.name` for correct
factory name casing.

Files changed: `src/bitwrench.js`
Tests: 12 tests in `test/bitwrench_test_new_apis.js`

### 12. bw.formData(el) -- DONE

Extracts form data as a plain object `{ fieldName: value }`. Handles text,
checkbox (boolean), radio, multi-select (array), textarea.

Files changed: `src/bitwrench.js`

### 13. bw.jsonPatch(obj, patches) -- DONE

RFC 6902 JSON Patch on plain objects. All 6 ops: add, remove, replace, move,
copy, test. JSON Pointer path parsing with ~0/~1 escaping. Mutates in place.
100% line coverage on the function.

Files changed: `src/bitwrench.js`, `src/bitwrench.d.ts`
Tests: 50 tests in `test/bitwrench_test_new_apis.js`

**Tests that exercise data-* as generic attributes (KEEP):**

## Candidates for Investigation (not committed)

These are worth thinking about but may not ship in v2.0.26.

### DOM morphing (opt-in)

`o.morph: true` on TACO options -- morph children on update instead of
replacing. Idiomorph is 2KB MIT, could be vendored. Helps the
prototyping-to-production transition where o.render kills focus/scroll.

Source: future-features.md #1.

### Accessibility audit

Audit all 50+ BCCL components against WAI-ARIA 1.2. Add missing roles,
aria-expanded, aria-haspopup, focus trap for modal/dropdown, keyboard
navigation for menus. Add `bw.contrastRatio()`, `bw.focusTrap()`,
`bw.announce()` (aria-live).

Source: future-features.md #3.

### Exit animations

`o.beforeRemove(el, done)` hook that delays removal until `done()` called.
View Transitions API integration in `bw.DOM()`.

Source: future-features.md #4.

### Flat component references (A2UI pattern)

Adjacency list input to bw.DOM() as alternative to nested TACO. Better for
LLM generation (less bracket matching), streaming, incremental updates.

Source: agui feedback #9.

### bwserve reconnection

Client-side reconnect with Last-Event-Id. Server replays or sends snapshot.

Source: agui feedback #10.

### State snapshot + delta for bwserve

`state_snapshot` + `state_delta` (JSON Patch) message types. More efficient
than full replace for IoT/embedded.

Source: agui feedback #6.

### aria-live on bwserve dynamic regions

Auto-add `aria-live="polite"` to containers receiving server-driven updates.

Source: agui feedback #4.
