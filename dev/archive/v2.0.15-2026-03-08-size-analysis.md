| generateAlternateCSS | 47 | Alternate palette CSS |
| deepMerge + updateTheme | 15 | Internal helpers |

**Key finding**: `defaultStyles` is 3,203 lines (64% of styles.js). This is a
single monolithic object containing ALL cosmetic CSS rules as JS property
strings. It cannot be tree-shaken.

The 27 `generate*Themed` functions (618 lines) are the per-component theme
generators that `generateThemedCSS` orchestrates. These are reasonably
compact (~23 lines each on average).

The 12 THEME_PRESETS are NOT large (13 lines total). Each is just 3 hex
color seeds + label + description. The theme *generator* produces the actual
CSS — the presets are inputs, not outputs. The earlier audit estimate of
"700 lines" was incorrect.

## Optimization Opportunities

### Tier 1: No breaking changes, immediate wins

| Item | Lines | Gzip |
|------|------:|-----:|
| Remove 5 dead Handle classes from components-v2.js | -393 | ~2.7KB |
| Remove duplicate color functions from bitwrench.js (import from color-utils instead) | -118 | ~0.4KB |
| Remove deprecated htmlTable/htmlTabs/selectTabContent/_attrsToStr | -164 | ~1.2KB |
| Remove deprecated getTheme/setTheme | -93 | ~0.7KB |
| **Subtotal** | **-768** | **~5.0KB** |

Post-Tier-1 estimated bundle: **~34KB gz** (from 39.3KB).

### Tier 2: Future architectural considerations

**BCCL "extended pack" model**: The embedded/ESP32 crowd wants BCCL but
doesn't need all 47 components. A future split could be:

- **bitwrench-core**: Runtime + CSS gen + reactivity + color (no make\*())
- **bitwrench** (default): Core + ~25 essential make\*() functions
- **bitwrench-extended**: Specialized components as opt-in addon
  (like bitwrench-code-edit today)

Candidates for "extended" (specialized, <10 usages each):
makeCarousel (205), makeChipInput (95), makeRange (97),
makeFileUpload (99), makePopover (94), makeStepper (63),
makeTimeline (69), makeMediaObject (62) = **784 lines** (~5KB gz)

**defaultStyles monolith**: The 3,203-line CSS object is the single largest
contributor to bundle size and cannot be tree-shaken. Future options:
1. Split into per-component style exports (users import what they need)
2. Generate on-demand from design tokens (only emit CSS for used components)
3. Move to external .css file (already exists as dist/bitwrench.css)

**File I/O section**: 965 lines (19% of bitwrench.js). The embedded crowd
rarely needs browser file save/load. Could be a separate optional module.

### What NOT to cut

- Theme presets are only 13 lines (12 x ~1 line of hex seeds). Negligible.
- The 27 `generate*Themed` functions (618 lines) are each compact (~23 lines
  avg) and necessary for palette-driven theming. No fat to trim.
- Color-utils (431 lines) is 100% used in the theming pipeline. Clean code.
