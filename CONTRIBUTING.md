# Contributing to Bitwrench

## Setup

Requires Node.js 18+ (tested on 18, 20, 22).

```bash
git clone https://github.com/deftio/bitwrench.git
cd bitwrench
npm install
```

## Development Workflow

1. Edit source files in `src/`
2. Build: `npm run build`
3. Test: `npm test`
4. Lint: `npm run lint`

## Directory Layout

| Directory | Purpose | Shipped to npm? |
|-----------|---------|-----------------|
| `src/` | Source files (v2) | Yes |
| `dist/` | Built distribution files (all formats) | Yes |
| `releases/v1/` | Frozen v1.2.16 archive | No |
| `releases/v2/` | Release snapshot (created by `build:release`) | No |
| `src_1x/` | Legacy v1 source (archived) | No |
| `test/` | Mocha unit tests | No |
| `tests/` | Playwright e2e tests | No |
| `tools/` | Build scripts | No |
| `dev/` | Design documents | No |
| `pages/` | Example/demo pages | No |

## Version Guarantee

`package.json` is the single source of truth for the version number. The chain is:

```
package.json  →  generate-version.cjs  →  src/version.js  →  rollup  →  dist/*.js  →  bw.getVersion()
```

**Never edit `src/version.js` directly.** It is auto-generated by `npm run generate-version` (which runs automatically before every build via `prebuild`).

CI tests enforce that `bw.getVersion().version === package.json.version`.

## Testing

- `npm test` — Unit tests with coverage (Mocha + c8)
- `npm run test:examples` — Playwright e2e tests for example pages
- `npm run test:all` — Run all test suites

## Code Rules

- **No direct DOM manipulation** in examples or library code. Use `bw.DOM()`, `bw.createDOM()`, and TACO patterns.
- **TACO format**: `{ t, a, c, o }` — Tag, Attributes, Content, Options.
- **CSS classes**: Use `bw_` prefix in JavaScript (e.g., `bw_card`). The library normalizes to `bw-` in HTML output.
- **Content escaping**: On by default. Use `bw.raw()` or `o: { raw: true }` to opt out.

## Build Commands

| Command | Description |
|---------|-------------|
| `npm run build` | Build all dist formats (UMD, ESM, CJS, ES5) |
| `npm run cleanbuild` | Clean build with index, API reference, and build manifest |
| `npm run build:release` | Snapshot dist/ into releases/v2/ |
| `npm run build:builds` | Generate dist/builds.json manifest |
| `npm run build:index` | Generate index.html using bitwrench |
| `npm run build:api-reference` | Generate API reference page |

## Release Process

1. Bump version in `package.json`
2. `npm run cleanbuild`
3. `npm test`
4. `npm run build:release`
5. Commit all changes
6. Tag: `git tag vX.Y.Z`
7. Push: `git push && git push --tags`
8. Publish: `npm publish`
