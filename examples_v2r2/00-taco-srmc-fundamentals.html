Introduction

A JavaScript UI Library

Bitwrench is a JavaScript library that represents HTML and CSS as plain JavaScript objects. Instead of writing HTML strings, you create UI using data structures called TACO (for HTML) and SRMC (for CSS). This approach requires no compilers, no JSX, no TSX, and no Babel - just pure JavaScript that runs directly in the browser.

The library provides a straightforward way to create dynamic user interfaces by treating UI elements as data that can be easily manipulated and updated.

Key Features

  • No build step required - write JavaScript, run JavaScript
  • HTML elements are represented as TACO objects
  • CSS styles are represented as SRMC objects
  • Built-in reactivity and dynamic templating
  • Direct DOM manipulation for efficient updates
  • Component system with lifecycle hooks
  • Works in modern browsers without transpilation

TACO Format

Tag - Attributes - Content - Options

TACO (Tag-Attributes-Content-Options) is how Bitwrench represents HTML elements as JavaScript objects. Each HTML element becomes a JavaScript object with up to four properties:

The TACO Structure

{
  t: "tag",        // HTML tag name (required)
  a: { ... },      // Attributes object (optional)
  c: "content",    // Content: string, array, or nested TACOs (optional)
  o: { ... }       // Options: lifecycle, state, etc. (optional)
}

Basic Examples

Simple Text

// TACO
{
  t: "p",
  c: "Hello, World!"
}

Result HTML

<p>Hello, World!</p>

With Attributes

// TACO
{
  t: "div",
  a: { 
    class: "card",
    id: "main-card"
  },
  c: "Card content"
}

Result HTML

<div class="card" id="main-card">
  Card content
</div>

Nested Elements

// TACO
{
  t: "div",
  a: { class: "card" },
  c: [
    { t: "h3", c: "Title" },
    { t: "p", c: "Content" },
    { 
      t: "button",
      a: { onclick: () => alert('Click!') },
      c: "Click Me"
    }
  ]
}

Result HTML

<div class="card">
  <h3>Title</h3>
  <p>Content</p>
  <button>Click Me</button>
</div>

Advanced Features

Options Object - Lifecycle & State

{
  t: "div",
  a: { class: "dynamic-component" },
  c: "I have lifecycle hooks!",
  o: {
    // Called when element is added to DOM
    mounted: (element) => {
      console.log('Component mounted:', element);
    },
    
    // Called when element is removed
    unmount: (element) => {
      console.log('Component unmounting');
    },
    
    // Component state
    state: {
      counter: 0,
      active: true
    }
  }
}

SRMC Format

Selector - Rules - Media - Children

SRMC (Selector-Rules-Media-Children) is how Bitwrench represents CSS as JavaScript objects. This allows you to generate and manipulate styles dynamically using data structures:

The SRMC Structure

{
  s: "selector",    // CSS selector (required)
  r: { ... },       // CSS rules object (optional)
  m: "@media ...",  // Media query (optional)
  c: [ ... ]        // Child SRMC objects (optional)
}

Basic Examples

Simple Style

// SRMC
{
  s: ".button",
  r: {
    padding: "10px 20px",
    backgroundColor: "#007bff",
    color: "white",
    borderRadius: "4px"
  }
}

Result CSS

.button {
  padding: 10px 20px;
  background-color: #007bff;
  color: white;
  border-radius: 4px;
}

With Media Query

// SRMC
{
  s: ".container",
  r: { maxWidth: "1200px" },
  m: "@media (max-width: 768px)",
  c: [{
    s: ".container",
    r: { maxWidth: "100%" }
  }]
}

Result CSS

.container {
  max-width: 1200px;
}

@media (max-width: 768px) {
  .container {
    max-width: 100%;
  }
}

How It Works

Comparing Approaches
Aspect Traditional Approach Bitwrench Approach
HTML Creation innerHTML = '<div>...</div>' bw.DOM(el, { t: 'div', ... })
Event Handling String onclick or addEventListener Direct function references in TACO
CSS Generation String concatenation SRMC objects with bw.css()
Build Process Often requires JSX, TSX, or Babel Pure JavaScript, no build step
Performance HTML parsing overhead Direct DOM manipulation
Dynamic Updates Replace entire HTML strings Update only changed elements

Three Rendering Modes

1. HTML String

bw.html(taco)

Returns HTML string for server-side rendering

2. DOM Creation

bw.createDOM(taco)

Creates DOM element without mounting

3. Live Mounting

bw.DOM(selector, taco)
