Fundamentals

What bitwrench is, why it exists, and how it works.

What Is Bitwrench?

Bitwrench is a JavaScript library for building web pages and applications. It lets you describe your user interface using plain JavaScript objects instead of writing HTML strings. You hand bitwrench an object, and it turns it into real elements on the page.

What you write (JavaScript)
bw.DOM('#app', {
  t: 'div',
  c: [
    { t: 'h1', c: 'Hello!' },
    { t: 'p', c: 'Built with bitwrench.' }
  ]
});
What appears on the page (HTML)
<div>
  <h1>Hello!</h1>
  <p>Built with bitwrench.</p>
</div>

That's it. No special syntax to learn, no compiler to set up, no build step to configure. You write JavaScript, bitwrench renders it. If you know basic JavaScript (objects, arrays, functions), you already know enough to use bitwrench.

The Idea Behind Bitwrench

UI as data

Every HTML element is really just an object: it has a tag name, some attributes, and some content. JavaScript has a native notation for exactly this kind of structure — object literals. Bitwrench uses that observation directly: describe your DOM as plain JavaScript objects, and let the library handle the rendering.

Once your UI is data, you can do everything JavaScript does well:

Transform

Use .map() to turn an array of records into a list of elements. Use .filter() to show a subset. The data drives the UI directly.

Compose

A function that returns a bitwrench object is a component. No special syntax, no registration — just functions calling functions.

Serialize

Bitwrench objects are JSON-compatible. A Python, Go, or Rust backend can emit JSON, and bitwrench renders it in the browser. Any language becomes a UI language.

Inspect

Because it's plain data, you can console.log() your UI, diff it, store it, send it over a network, or test it without a browser.

Where this approach shines

Prototyping — no project setup, no build tools. Include one script tag and go.

Embedded devices — ESP32, Raspberry Pi, and other IoT boards can serve bitwrench objects as JSON. The device sends data; the browser renders the UI.

Polyglot backends — any server that can produce JSON can generate bitwrench UIs. No JavaScript templating engine required on the server.

Learning and teaching — the gap between "I know JavaScript objects" and "I can build a web UI" is one function call: bw.DOM().

The TACO Format

Tag - Attributes - Content - Options

TACO stands for Tag, Attributes, Content, Options. It's a plain JavaScript object that describes an HTML element. Every HTML element has a tag name, some attributes, and some content - a TACO object mirrors this structure exactly.

The TACO structure
{
  t: 'div',            // Tag: what kind of element (div, p, button, etc.)
  a: { class: 'card' }, // Attributes: id, class, style, event handlers
  c: 'Hello world',    // Content: text, another TACO, or an array of TACOs
  o: { ... }           // Options: lifecycle hooks, state (advanced, optional)
}

Only t (tag) is required. Everything else is optional. Let's build up from simple examples:

A paragraph

TACO
{ t: 'p', c: 'Hello, world!' }
Resulting HTML
<p>Hello, world!</p>

An element with attributes

TACO
{
  t: 'a',
  a: {
    href: 'https://example.com',
    class: 'link',
    target: '_blank'
  },
  c: 'Visit Example'
}
Resulting HTML
<a href="https://example.com"
   class="link"
   target="_blank">
  Visit Example
</a>

Nested elements

Content (c) can be a string, another TACO object, or an array of TACOs. This is how you nest elements inside each other:

TACO
{
  t: 'div',
  a: { class: 'card' },
  c: [
    { t: 'h3', c: 'Card Title' },
    { t: 'p', c: 'Some content.' },
    { t: 'button', c: 'Click me' }
  ]
}
Resulting HTML
<div class="card">
  <h3>Card Title</h3>
  <p>Some content.</p>
  <button>Click me</button>
</div>

Adding interactivity

Event handlers like onclick go in the attributes object, just like they would in HTML - except they're real JavaScript functions, not strings: 

{
  t: 'button',
  a: {
    class: 'bw-btn bw-btn-primary',
    onclick: function() { alert('You clicked!'); }
  },
  c: 'Click Me'
}

Why this matters

Since a TACO is just a JavaScript object, you can build it with if statements, .map() arrays into lists, spread objects together, pass them as function arguments, store them in variables, and use every tool JavaScript gives you. No special template syntax needed.

Rendering TACOs

Three ways to turn objects into UI

Bitwrench gives you three functions for turning TACO objects into output, depending on what you need:

bw.DOM(selector, taco)

Renders a TACO into an existing element on the page. This is the most common way to use bitwrench - it replaces the content of the selected element.

bw.html(taco)

Returns an HTML string. Useful for server-side rendering, email templates, or any time you need HTML as text rather than live DOM elements.

bw.createDOM(taco)

Creates a real DOM element without putting it on the page. Use this when you want to build elements first and insert them later.

The Bitwrench Pattern

State, Render, Mount

Interactive applications follow a simple three-step pattern:

  1. State - a plain JavaScript object holding your data
  2. Render - a function that builds a TACO from that state
  3. Mount - bw.DOM() puts it on the page

When something changes (a button click, new data from a server), you update the state and call render again. Bitwrench replaces the old content with the new. That's the entire architecture.

Live example: a counter

No magic

There's no virtual DOM, no reactivity system, no observable patterns. You update a variable and call a function. The simplicity is the point - you can understand the entire flow by reading the code.

Data-Driven UI

JavaScript is the template language

Because TACOs are JavaScript objects, you can use .map() to turn arrays of data into arrays of elements. This is the most powerful idea in bitwrench - your data drives your UI directly: 

var fruits = ['Apple', 'Banana', 'Cherry'];

bw.DOM('#list', {
  t: 'ul',
  c: fruits.map(function(fruit) {
    return { t: 'li', c: fruit };
  })
});

The list has three items because the array has three elements. Add a fourth element and re-render - you get four items. The data is the source of truth.

Generating CSS

Styles as JavaScript objects too

Just as TACO represents HTML as objects, bitwrench can generate CSS from JavaScript objects using bw.css(). This lets you create dynamic styles that respond to data:

JavaScript
var css = bw.css({
  '.card': {
    padding: '1rem',
    borderRadius: '8px',
    backgroundColor: '#fff'
  },
  '.card h3': {
    color: '#006666'
  }
});

bw.injectCSS(css);
Generated CSS
.card {
  padding: 1rem;
  border-radius: 8px;
  background-color: #fff;
}

.card h3 {
  color: #006666;
}

This is particularly useful for theming - you can define color palettes as variables and generate an entire stylesheet dynamically. See the Themes & Styling example for a full demo.
