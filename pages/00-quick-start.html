Quick Start

From zero to working UI in 2 minutes. Code first, explanations after.

Your First Bitwrench Page

This is the complete code for a bitwrench page. One script tag, one function call: 

<!DOCTYPE html>
<html>
<head>
  <script src="https://unpkg.com/bitwrench@2/dist/bitwrench.umd.js"></script>
  <link rel="stylesheet" href="https://unpkg.com/bitwrench@2/dist/bitwrench.css">
</head>
<body>
  <div id="app"></div>
  <script>
    bw.DOM('#app', { t: 'h1', c: 'Hello bitwrench!' });
  </script>
</body>
</html>

Building Blocks

Every HTML element is a JavaScript object with a tag, attributes, and content. Build up progressively:

This is called TACO

Tag, Attributes, Content, Options. You've just used it. The name comes after you've seen the idea — it's just a plain JavaScript object that maps directly to HTML. Only t is required; everything else is optional.

Where Do Styles Come From?

Bitwrench includes built-in CSS classes (bw-btn, bw-card, etc.) that give you Bootstrap-like styling. You get them by including <link href="bitwrench.css"> in your page.

Helpers return JavaScript objects, not HTML

bw.makeButton() returns a TACO object. bw.DOM() renders it. The helper is just a shortcut for writing the object yourself. See the Styling page for the full CSS story.

Making It Interactive

When state changes, call render again. Bitwrench replaces the old content with the new. That's the entire update model.

Honest trade-off

This is manual re-rendering. React and Vue do this automatically via hooks and reactivity. Bitwrench trades convenience for simplicity — no virtual DOM, no magic, just "call render again." You always know exactly when and why your UI updates.

Data-Driven UI

This is a genuine strength: your data IS your UI. No template directives, no v-for, no {'{#each}'}. Just .map().
