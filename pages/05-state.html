State & Interactivity

How bitwrench manages state, from simple closures to component factories and pub/sub.

Bitwrench's Approach to State

Frameworks like React, Solid, and Svelte each have their own approach to state management — virtual DOMs, reactive signals, compiler-generated update code. They work well for the problems they were designed to solve.

Bitwrench takes a different approach. State is a plain JavaScript object stored directly on the DOM element it belongs to. Updates are explicit function calls rather than automatic reactions. There is no virtual DOM, no dependency tracking, no compiler step. A component re-renders when — and only when — bw.update(el) is called.

Coming from another framework?

Concept How bitwrench does it
Component (JSX / .svelte file)Factory function returning a TACO object

Context / stores / global signalsbw.pub() / bw.sub() (application-scoped pub/sub)
useRefThe element itself — state and DOM are the same object

The tradeoff: bitwrench requires the developer to say when to update, rather than detecting changes automatically. In return, every state change is an explicit function call you can see and step through in a debugger. The sections below walk through each pattern from simplest to most complex.

The toolkit at a glance

o.state — initial state object, stored as el._bw_state on mount
o.render(el) — render function, stored as el._bw_render, called on mount and on bw.update()
bw.update(el) — re-run the render function and emit bw:statechange
bw.patch(id, content) — targeted update of a single element (no full re-render)
bw.pub(topic, detail) / bw.sub(topic, handler) — decoupled cross-component messaging
o.mounted(el) — lifecycle hook, runs after the element is in the DOM
o.unmount(el) — cleanup hook, runs when the element is removed

Memory and cleanup

When bw.DOM() replaces an element's children, the old DOM nodes and any event listeners attached to them are removed by the browser's garbage collector. There is no manual listener cleanup step — replacing the DOM is the cleanup.

For resources that live outside the DOM (timers, WebSocket connections, pub/sub subscriptions), use the o.unmount(el) lifecycle hook. Bitwrench calls it automatically via bw.cleanup() before replacing content. Pub/sub subscriptions can also be tied to an element's lifecycle by passing the element to bw.sub() — when the element is cleaned up, its subscriptions are removed automatically.

Debugging

All component state is visible in the browser's built-in inspector. Select any bitwrench component in the Elements panel, then type $0._bw_state in the console to see its current state object. You can modify it directly and call bw.update($0) to see the result. No browser extension required.

The Simplest State Pattern

The most basic approach: a closure variable and a render function. When state changes, call render again. This works well for simple, standalone components.

How State Works in Bitwrench

In bitwrench, you build components by writing functions that return plain JavaScript objects describing what to render. State lives on the DOM element itself. When an element is removed from the page, its state goes with it. When you create three instances of a component, each one carries its own independent state.

1. Define o: { state, render }
→
2. Mount render(el) auto-called
→
3. Update bw.update(el)

The pattern

A component factory returns a JavaScript object with o.state and o.render. When bitwrench creates the DOM element, it stores state as el._bw_state and the render function as el._bw_render, then calls render(el) automatically once the element is in the DOM. To re-render, call bw.update(el) — it re-runs el._bw_render(el) and emits a bw:statechange event so other components can react.

Why state lives on the element

Storing state on el._bw_state ties the state to the element's lifecycle. Delete the element → state is gone. Create three instances of a component → each has its own state on its own element. It's a simple model: the DOM element is the component.

The complete pattern
// 1. Factory function — returns an object with TACO + API
function makeCounter(opts) {
  var id = bw.uuid('ctr');

  var taco = {
    t: 'div', a: { id: id },
    o: {
      state: { count: opts.start || 0 },  // initial state
      render: function(el) {               // stored as el._bw_render
        var s = el._bw_state;              // read state from the element
        bw.DOM(el, {                       // re-render into self
          t: 'div', a: { style: 'background: var(--bw-card-bg, #fff); '
            + 'border: 1px solid var(--bw-card-border, #dee2e6); '
            + 'border-radius: 8px; padding: 1rem; '
            + 'text-align: center;' },
          c: [
            { t: 'h4', c: opts.label },
            { t: 'div', c: String(s.count) },
            bw.makeButton({ text: '+',
              onclick: function() {
                s.count++;
                bw.update(el);             // re-renders + emits event
              }
            })
          ]
        });
      }
    }
  };

  // 2. Return TACO + getter/setter API
  return {
    taco: taco, id: id,
    getCount: function() { /* read el._bw_state */ },
    setCount: function(n) { /* write + bw.update(el) */ }
  };
}

// 3. Use it — each instance is independent
var a = makeCounter({ label: 'A', start: 0 });
var b = makeCounter({ label: 'B', start: 10 });
bw.DOM('#app', {
  t: 'div', c: [a.taco, b.taco]
});
What happens at runtime
// bw.createDOM processes the TACO:
//   1. Creates <div id="ctr-abc123">
//   2. Stores state: el._bw_state = { count: 0 }
//   3. Stores render: el._bw_render = o.render
//   4. Queues render via requestAnimationFrame
//
// After the element is in the DOM:
//   5. el._bw_render(el) fires automatically
//   6. render reads el._bw_state.count (= 0)
//   7. bw.DOM(el, newContent) replaces children
//      but preserves el._bw_state and el._bw_render
//
// On button click:
//   8. s.count++ modifies el._bw_state.count
//   9. bw.update(el) calls el._bw_render(el)
//  10. bw.update also emits bw:statechange
//      so other components can listen and react
//
// If the element is removed:
//  11. el._bw_state goes away with the element
//  12. No cleanup needed — nothing external to clean up
//
// Key insight:
//   - opts (label, start) is configuration — fixed at creation
//   - state (count) is mutable — lives on the element
//   - bw.update(el) = re-render + notify listeners

Encapsulated Components

Three counters from the same factory. Each has its own state on its own DOM element. Click any counter — the others don't change. The factory returns a getter/setter API alongside the TACO, enabling cross-component coordination (next section).

Source
function makeCounter(opts) {
  var id = bw.uuid('ctr');
  var name = opts.label;

  // Inline styles — portable, no external CSS needed
  var cardStyle = 'background: var(--bw-card-bg, #fff); '
    + 'border: 1px solid var(--bw-card-border, #dee2e6); '
    + 'border-radius: 8px; padding: 1rem 1.25rem; '
    + 'text-align: center; min-width: 160px; '
    + 'flex: 1 1 160px;';
  var valueStyle = 'font-size: 2rem; font-weight: 700; '
    + 'color: var(--bw-teal-dark, #00796b); '
    + 'margin: 0.5rem 0;';
  var btnsStyle = 'display: flex; gap: 0.35rem; '
    + 'justify-content: center; flex-wrap: wrap;';

  var taco = {
    t: 'div', a: { id: id },
    o: {
      state: { count: opts.start || 0 },
      render: function(el) {
        var s = el._bw_state;
        bw.DOM(el, {
          t: 'div', a: { style: cardStyle },
          c: [
            { t: 'h4', a: { style: 'margin: 0 0 0.5rem;' },
              c: name },
            { t: 'div', a: { style: valueStyle },
              c: String(s.count) },
            { t: 'div', a: { style: btnsStyle }, c: [
              bw.makeButton({ text: '+',
                variant: 'primary',
                onclick: function() {
                  s.count++; bw.update(el);
                }
              }),
              bw.makeButton({ text: '\u2013',
                variant: 'secondary',
                onclick: function() {
                  s.count--; bw.update(el);
                }
              }),
              bw.makeButton({ text: 'Reset',
                variant: 'secondary',
                onclick: function() {
                  s.count = opts.start || 0;
                  bw.update(el);
                }
              })
            ]}
          ]
        });
      }
    }
  };

  return {
    taco: taco, name: name, id: id,
    getCount: function() {
      var el = document.getElementById(id);
      return el && el._bw_state
        ? el._bw_state.count : null;
    },
    setCount: function(n) {
      var el = document.getElementById(id);
      if (el && el._bw_state) {
        el._bw_state.count = n;
        bw.update(el);
      }
    }
  };
}

var cardA = makeCounter(
  { label: 'Visitors', start: 42 });
var cardB = makeCounter(
  { label: 'Orders',   start: 7 });
var cardC = makeCounter(
  { label: 'Errors',   start: 0 });
var allCards = [cardA, cardB, cardC];

bw.DOM('#counters', {
  t: 'div',
  a: { style: 'display: flex; gap: 1rem; '
     + 'flex-wrap: wrap;' },
  c: allCards.map(function(c) {
    return c.taco;
  })
});

Cross-Component Coordination

The global panel reads and modifies all three counters via the getter/setter API. It auto-updates whenever any counter changes, because bw.update() emits a bw:statechange event that bubbles up. The global panel listens with bw.on() — no tight coupling.

Source
// Listen for statechange from any counter
bw.on('#counter-demo', 'statechange',
  function() { renderGlobal(); }
);

function renderGlobal() {
  var counts = allCards.map(function(c) {
    return c.getCount() || 0;
  });
  var sum = counts.reduce(
    function(a, b) { return a + b; }, 0
  );
  var breakdown = allCards.map(function(c, i) {
    return c.name + '(' + counts[i] + ')';
  }).join(' + ');

  bw.DOM('#global-demo', {
    t: 'div', a: { class: 'global-panel' },
    c: [
      { t: 'h4', c: 'Cross-Component Controls' },
      { t: 'div', a: { class: 'global-btns' }, c: [
        bw.makeButton({ text: 'Inc All (+1)',
          variant: 'primary',
          onclick: function() {
            allCards.forEach(function(c) {
              c.setCount(c.getCount() + 1);
            });
            renderGlobal();
          }
        }),
        bw.makeButton({ text: 'Dec All (\u20131)',
          variant: 'primary',
          onclick: function() {
            allCards.forEach(function(c) {
              c.setCount(c.getCount() - 1);
            });
            renderGlobal();
          }
        }),
        bw.makeButton({ text: 'Reset All',
          variant: 'secondary',
          onclick: function() {
            allCards.forEach(function(c) {
              c.setCount(0);
            });
            renderGlobal();
          }
        })
      ]},
      { t: 'div', a: { class: 'sum-display' },
        c: breakdown + ' = ' + sum }
    ]
  });
}

Targeted Updates: bw.patch()

bw.patch(id, content) updates a single element by its UUID without rebuilding the entire component tree. Use it for value-only changes (counters, labels). Use bw.update() for structural changes (add/remove items).

Source
// Each list item has a stable UUID for its value
var item = {
  id:    bw.uuid('item'),   // row UUID
  valId: bw.uuid('val'),    // value UUID
  name:  'Counter #1',
  count: 0
};

// Value change: targeted, no rebuild
bw.makeButton({ text: '+',
  onclick: function() {
    item.count++;
    bw.patch(item.valId, String(item.count));
  }
});

// Structural change: full re-render
function addItem() {
  listItems.push(newItem);
  renderList();   // rebuilds via bw.DOM()
}

Event Systems: emit/on & pub/sub

Bitwrench provides two event systems. emit/on wraps DOM CustomEvents (scoped to an element subtree, bubbles up). pub/sub is application-scoped (topic-based, decoupled from the DOM tree). Both coexist.

Comparison

Aspect emit/on pub/sub
ScopeDOM subtree (bubbles)Application-wide (topic)
Trigger1:1 with DOM actionComposite / logic
CleanupAutomatic (DOM GC)unsub() or element-tied
CouplingSpatial (ancestor)None (any component)
ErrorNative DOM throwsTry/catch, log, continue
Source
// Publish a message to a topic
bw.pub('notification', { text: msg });

// Subscribe — returns unsub function
var unsub = bw.sub('notification',
  function(detail) {
    messages.push(detail.text);
    renderPanel();
  }
);

// Later: remove the subscription
unsub();

Stateful Component: Todo List

A more complex component with array state, conditional rendering, and multiple interactions. The same pattern scales: state on the element, o.render reads it, bw.update(el) re-renders.

Source
function makeTodoList(opts) {
  var id = bw.uuid('todo');

  var todoItem = function(item, el) {
    var s = el._bw_state;
    return {
      t: 'div',
      a: { class: 'todo-item'
        + (item.done ? ' done' : '') },
      c: [
        { t: 'input', a: {
          type: 'checkbox',
          checked: item.done ? 'checked' : undefined,
          onchange: function() {
            item.done = !item.done;
            bw.update(el);
          }
        }},
        { t: 'span', c: item.text },
        bw.makeButton({
          text: '\u00D7', variant: 'secondary',
          size: 'sm',
          onclick: function() {
            s.todos = s.todos.filter(function(t) {
              return t.id !== item.id;
            });
            bw.update(el);
          }
        })
      ]
    };
  };

  function addTodo(el) {
    var s = el._bw_state;
    var input = bw.$('#' + id
      + ' input[type="text"]')[0];
    if (!input || !input.value.trim()) return;
    s.todos.push({
      id: s.nextId++,
      text: input.value.trim(),
      done: false
    });
    bw.update(el);
  }

  var initial = (opts.initial || []).map(
    function(text, i) {
      return { id: i + 1, text: text, done: false };
    }
  );

  return {
    t: 'div', a: { id: id },
    o: {
      state: {
        todos: initial,
        nextId: initial.length + 1
      },
      render: function(el) {
        var s = el._bw_state;
        var remaining = s.todos.filter(
          function(t) { return !t.done; }
        ).length;

        bw.DOM(el, { t: 'div', c: [
          { t: 'div', a: { class: 'todo-input-row' },
            c: [
            { t: 'input', a: {
              type: 'text',
              placeholder: opts.placeholder
                || 'Add item...',
              onkeydown: function(e) {
                if (e.key === 'Enter') addTodo(el);
              }
            }},
            bw.makeButton({
              text: 'Add', variant: 'primary',
              onclick: function() { addTodo(el); }
            })
          ]},
          { t: 'div', c: s.todos.map(function(item) {
            return todoItem(item, el);
          })},
          { t: 'div', a: {
            style: 'margin-top:0.5rem; '
              + 'font-size:0.8125rem; '
              + 'color:var(--bw-text-secondary);'
            },
            c: remaining + ' of '
              + s.todos.length + ' remaining'
          }
        ]});
      }
    }
  };
}

bw.DOM('#todo', makeTodoList({
  initial: ['Read the docs',
    'Build a prototype', 'Ship it'],
  placeholder: 'What needs doing?'
}));

Dynamic CSS Generation

bw.css() converts JavaScript objects to CSS strings. bw.injectCSS() inserts them into the document. Together they let you generate entire stylesheets at runtime from data.

Source
function makeThemeSwitcher() {
  var themes = {
    ocean:  { bg: '#e0f7fa', fg: '#004d40', accent: '#00796b' },
    sunset: { bg: '#fff3e0', fg: '#bf360c', accent: '#e65100' },
    forest: { bg: '#e8f5e9', fg: '#1b5e20', accent: '#2e7d32' }
  };
  var id = bw.uuid('theme');

  function render(el) {
    var s = el._bw_state;
    var t = themes[s.current];

    // Generate CSS from a JS object and inject it
    bw.injectCSS(bw.css({
      ['#' + id + ' .themed']: {
        background: t.bg, color: t.fg,
        'border-color': t.accent,
        padding: '1.25rem', 'border-radius': '8px',
        border: '2px solid', transition: 'all 0.3s'
      },
      ['#' + id + ' .themed h4']: {
        color: t.accent, margin: '0 0 0.35rem'
      }
    }), { id: id + '-css' });

    bw.DOM(el, { t: 'div', c: [
      { t: 'div',
        a: { style: 'display:flex; gap:0.5rem; margin-bottom:0.75rem;' },
        c: Object.keys(themes).map(function(key) {
          return bw.makeButton({
            text: key.charAt(0).toUpperCase() + key.slice(1),
            variant: key === s.current ? 'primary' : 'secondary',
            onclick: function() {
              s.current = key; render(el);
            }
          });
        })
      },
      { t: 'div', a: { class: 'themed' }, c: [
        { t: 'h4', c: s.current.charAt(0).toUpperCase()
          + s.current.slice(1) + ' Theme' },
        { t: 'p', a: { style: 'margin:0; font-size:0.875rem;' },
          c: 'Colors generated from a JS object via bw.css().' }
      ]}
    ]});
  }

  return {
    t: 'div', a: { id: id },
    o: {
      state: { current: 'ocean' },
      mounted: function(el) { render(el); }
    }
  };
}

bw.DOM('#themes', makeThemeSwitcher());

Performance

Bitwrench renders directly to the DOM. This test creates and mounts 1,000 elements in a single bw.DOM() call.

Two update strategies

bw.DOM() replaces all children — straightforward for full re-renders. But updating one value in a list of 1,000 items doesn't require rebuilding the other 999. That's what bw.patch(id, value) is for: it finds a single element by UUID and updates its content directly. Use bw.DOM() for structural changes (add/remove items) and bw.patch() for value changes (counters, labels, status text). The dynamic list above demonstrates both in combination.

Source
function makePerfTest() {
  var id = bw.uuid('perf');

  function run(el) {
    var start = performance.now();
    var items = [];
    for (var i = 0; i < 1000; i++) {
      items.push({
        t: 'span',
        a: { class: 'bw-badge bw-badge-primary',
             style: 'margin:2px;' },
        c: 'Item ' + (i + 1)
      });
    }
    bw.DOM('#' + id + '-items', { t: 'div', c: items });
    var ms = (performance.now() - start).toFixed(2);
    bw.DOM('#' + id + '-time', {
      t: 'div', a: { class: 'bw-alert bw-alert-success' },
      c: 'Rendered 1,000 items in ' + ms + ' ms'
    });
  }

  return {
    t: 'div', a: { id: id },
    o: {
      mounted: function(el) {
        bw.DOM(el, {
          t: 'div', c: [
            bw.makeButton({ text: 'Render 1,000 Items',
              variant: 'primary',
              onclick: function() { run(el); }
            }),
            { t: 'div', a: { id: id + '-time',
              style: 'margin-top:0.75rem;' }, c: '' },
            { t: 'div', a: { id: id + '-items',
              style: 'margin-top:0.75rem; max-height:200px; '
                + 'overflow-y:auto;' }, c: '' }
          ]
        });
      }
    }
  };
}

bw.DOM('#perf', makePerfTest());
